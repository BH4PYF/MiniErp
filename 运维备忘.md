## 服务器总体架构

```
                    宿主机 Nginx (:80/:443)
                    /                    \
     erp.sdyhjzgc.com                  sdyhjzgc.com
     proxy_pass ↓                      fastcgi_pass ↓
     Gunicorn :6666 (Docker)           PHP-FPM :9000 (Docker)
          ↓
     PostgreSQL :5432 (Docker)         MariaDB :3306 (宿主机)
     Redis :6379 (Docker)
     Celery (Docker)
```

两个项目共用宿主机 Nginx，通过 `server_name` 区分。

## 关键变更记录

### 2026-05-15 — 数据库切换

**问题**: `postgres-db` Docker 容器崩溃（exit 255），宿主机 PostgreSQL 16 自动接管了 5432 端口。

**操作**:
```bash
# 停止并禁用宿主机 PostgreSQL
sudo systemctl stop postgresql@16-main
sudo systemctl disable postgresql@16-main

# 重新拉起 Docker 数据库容器
docker compose up -d db
```

### 2026-05-15 — Nginx 恢复

**问题**: `erp.sdyhjzgc.com` 无响应，根因是宿主机 Nginx 服务未运行。

**操作**:
```bash
sudo systemctl start nginx
sudo systemctl enable nginx
```

### 2026-05-15 — Nginx 看门狗部署

新增 `scripts/nginx_watchdog.sh`，每分钟检查 Nginx 存活，异常自动拉起。

```bash
# Crontab 条目
* * * * * /home/abc/MiniErp/scripts/nginx_watchdog.sh
```

**三重保护**:
1. `systemd Restart=always` — 进程崩溃 5s 自动重启
2. cron 看门狗 — 每分钟拉起
3. `systemctl enable nginx` — 开机自启

## 常用运维命令

### 服务管理
```bash
# MiniErp
cd /home/abc/MiniErp
docker compose up -d          # 启动所有服务
docker compose ps              # 查看状态
docker compose logs -f web     # 查看 Web 日志
docker compose restart web     # 重启 Web

# Nginx
sudo systemctl status nginx
sudo systemctl restart nginx
sudo nginx -t                  # 配置检查
```

### 健康检查
```bash
curl -s http://127.0.0.1:6666/health/           # Gunicorn
curl -sk https://erp.sdyhjzgc.com/health/       # 通过域名
pg_isready -h 127.0.0.1 -p 5432                  # PostgreSQL
docker ps --format "{{.Names}} {{.Status}}"      # Docker 概览
```

## Nginx 站点配置

MiniErp: `/etc/nginx/sites-enabled/erp.sdyhjzgc.com.conf`
- HTTP 80 → HTTPS 443 重定向
- `/static/` → `/home/abc/MiniErp/staticfiles/`
- `/media/` → `/home/abc/MiniErp/media/`
- `/` → proxy_pass `http://127.0.0.1:6666`

SSL 证书: `/etc/nginx/ssl/erp.sdyhjzgc.com_fullchain.crt` + `.key`

## 端口冲突预防

宿主机 PostgreSQL 和 Redis 均已禁用自启：
```bash
sudo systemctl disable --now postgresql@16-main
sudo systemctl mask postgresql
```

## 故障排查速查

| 症状 | 检查步骤 |
|------|----------|
| 域名无响应 | `systemctl status nginx` → `sudo nginx -t` → `ss -tlnp \| grep ':80\|:443'` |
| 502 Bad Gateway | `docker ps \| grep minierp-web` → `curl localhost:6666/health/` |
| 数据库连接错误 | `pg_isready -h 127.0.0.1 -p 5432` → `docker ps \| grep postgres` |
| 页面加载慢 | `docker stats` → `free -h` → `df -h` |
